# PK版吃什么：技术实现方案 🍱

## 📝 1. 项目简介
这是一个基于 **React** 开发的纯前端 Web 应用，旨在通过有趣的 1vs1 竞技（PK）模式，帮助用户在 3 分钟内解决“吃什么”的世纪难题。

## 🚀 2. 技术选型
- **前端框架**: `React` (使用 `Vite` 构建，保证极致加载速度)
- **动画库**: `Framer Motion` (提供弹簧物理动画，实现点击、晋级时的“极致解压感”)
- **分享技术**: `html-to-image` (将生成的对战表转化为朋友圈适配的海报)
- **数据持久化**: `LocalStorage` (用户自定义的美食数据保存在本地浏览器)
- **部署平台**: `Vercel`

## 🛠️ 3. 开发环境要求
为了确保应用能够顺利运行和构建，请确保你的本地环境已安装以下工具：
- **Node.js**: 建议版本为 `v18.x` 或更高版本。
- **包管理器**: `npm` (Node 自带) 或 `pnpm` (推荐，更快)。
- **Git**: 用于版本管理及与 Vercel 同步。
- **现代化浏览器**: Chrome, Edge 或 Safari（开发与调试使用）。

## 🕹️ 4. 核心逻辑与功能
### 3.1 PK 对决流程
- **规则**: 采用淘汰赛制（16进8 -> 8进4 -> 4进2 -> 决赛）。
- **自动化**: 默认 3 秒倒计时。若用户未点击，系统**随机选出**一个进入下一轮（确保决策速度）。
- **灵活性**: 用户可调整 PK 总人数（2/4/8/16/32）和倒计时时长。

### 3.2 视觉风格 (UI/UX)
- **调性**: 色彩明亮、酷炫、高对比度。
- **关键词**: 动态模糊、缩放回弹、霓虹光效。
- **布局**: 适配 PC 与 手机端，移动端优化大点击区域。

### 3.3 全局美食库
- **内置数据**: 通过 AI 生成风格高度统一的 **3D 质感美食插画**（如：发光的汉堡、清新的寿司）。
- **自定义**: 用户可手动添加美食（名称+自选配图）。

## 📸 5. 社交分享设计
- **海报内容**: 
  - 顶部的夸张**冠军宣言**。
  - 完整的**晋级战报大表**（展示每一轮的胜出路径）。
  - 个性化的背景渐变与 Logo。
- **导出规格**: 适配手机竖版的 9:16 或 3:4 比例，确保朋友圈显示不被裁切。

## 📅 6. 开发里程碑
1. **Phase 1**: 基础架构搭建 & 静态 UI 实现。
2. **Phase 2**: PK 核心算法逻辑 & Framer Motion 动画注入。
3. **Phase 3**: 美食库管理与 LocalStorage 适配。
4. **Phase 4**: 分享海报生成功能开发与整体视觉打磨。
